Étape 2: les classifieurs¶

Un classifieur (binaire dans ce cours) permet, pour chaque point d'entre donn, de calculer un score (qui est une valeur de $\mathbb{R}$). Si le score est strictement ngatif, la donne est associe la classe $-1$, sinon la classe $+1$.

Représentation orientée objets¶

Pour implmenter nos algorithmes d'apprentissage et pour reprsenter les donnes, nous allons utiliser quelques lments de programmation oriente objet (POO) qu'autorise le langage Python.

Si vous ne connaissez rien la POO, il est fortement conseill de lire une documentation ou suivre un tutorial sur le web (par exemple, http://apprendre-python.com/page-apprendre-programmation-orientee-objet-poo-classes-python-cours-debutants).

La classe Classifier¶

Les classifieurs que l'on va implmenter seront dfinis l'aide d'une classe qui va driver de la classe Classifier suivante. Cette classe est une classe "abstraite", elle sera drive par hritage pour dfinir des classifieurs particuliers (knn, arbres de dcision, etc.).

Cette classe contient 5 mthodes:

  • __init__: qui permet d'initialiser les paramtres du classifieur. Les classes filles tendront cette mthode en rajoutant les paramtres qui les concernent. Le paramtre obligatoire d'un classifieur est le nombre de dimensions des donnes qu'il aura traiter.
  • train: c'est la mthodes qui permet d'entraner le modle, on doit donc donner en argument le dataset qui sert l'entranement.
  • score: cette mthode est utilise une fois le classifieur entrain. Elle prend en argument la description d'un exemple et elle rend un score (une valeur de $\mathbb{R}$).
  • predict: cette mthode, comme la prcdente, s'utilise une fois le classifieur entran, elle donne le label (-1 ou bien +1) qui est prdit par le classifieur pour la description d'exemple donne en argument. Gnralement, cette valeur de prdiction s'obtient partir du score: si le score est strictement ngatif la prdiction vaut $-1$, sinon elle vaut $+1$.
  • accuracy: comme les 2 prcdentes, cette mthode s'utilise une fois le classifieur entran. Elle permet de calculer la qualit du classifieur sur un dataset donn, cette qualit est le taux de bonne classification du classifieur sur le dataset. C'est donc une valeur de $[0,1]$ qui s'obtient divisant le nombre d'exemples du dataset qui sont bien classs par le classifieur par le nombre total d'exemples du dataset.

Les mthodes train, score et predict ne seront pas dfinies dans la classe Classifier mais dans les classes qui hriteront ce cette classe car ces mthodes dpendent du modle que l'on choisira d'implmenter.

Par contre, la fonction accuracy renvoie le pourcentage de points bien classs. Elle peut tre dfinie ds maintenant dans cette classe Classifier.

def plot_frontiere(desc_set, label_set, classifier, step=30):
    """ desc_set * label_set * Classifier * int -> NoneType
        Remarque: le 4e argument est optionnel et donne la "rsolution" du trac
        affiche la frontire de dcision associe au classifieur
    """
    mmax=desc_set.max(0)
    mmin=desc_set.min(0)
    x1grid,x2grid=np.meshgrid(np.linspace(mmin[0],mmax[0],step),np.linspace(mmin[1],mmax[1],step))
    grid=np.hstack((x1grid.reshape(x1grid.size,1),x2grid.reshape(x2grid.size,1)))
    
    # calcul de la prediction pour chaque point de la grille
    res=np.array([classifier.predict(grid[i,:]) for i in range(len(grid)) ])
    res=res.reshape(x1grid.shape)
    # tracer des frontieres
    plt.contourf(x1grid,x2grid,res,colors=["red","cyan"],levels=[-1000,0,1000])
